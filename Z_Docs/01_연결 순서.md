1. 클라이언트 -> 서버, 파이프 생성 시도
	- 서버, 클라이언트 추가 시도
		- 연결 풀 초과 시 Dispose
2. 서버 -> 클라이언트, 이름과 UUID 요청
	- 클라이언트에 저장된 이름이 있을 경우
		1. 클라이언트 -> 서버, 이름 전달
		2. 서버, 이름 확인
			- 이 이름으로 이미 접속이 되어있을 경우 
				- 서버 -> 클라이언트, 이름 재요청
				- 사용자 이름 입력 순서로...
		3. 서버, 유저 목록에 해당 클라이언트 등록
		4. 서버 -> 클라이언트, UUID 전달, 방 목록 전달
	- 없을 경우
		1. 클라이언트, 사용자의 이름 입력
		2. 클라이언트 -> 서버, 이름만 전송
		3. 서버, 이름 확인
			- 이름 중복 있을 경우
				- 서버 -> 클라이언트, 이름 재요청
				- 사용자 이름 입력 순서로..
		4. 서버, 유저 목록에 해당 클라이언트 등록
		5. 서버 -> 클라이언트, UUID 전달, 방 목록 전달
3. 클라이언트 -> 서버, 방 입장 요청
	- 지금은 방이 하나니까 2번 전달 받고 바로 0번방 요청
4. 서버, 방 상태 확인
	- 밴 리스트에 있거나(예정), 방이 포화상태일 경우 입장 거절
5. 서버 -> 클라이언트 유저 목록 전달
	1. 클라이언트, 유저 목록 매핑 등록
	2. 클라이언트, 유저 목록 출력
6. 채팅 시작
